# AI智能体与人工审阅结合工作框架资源包整理计划

## 1. 资源包目录结构设计

基于框架文档内容，设计6个核心子目录，确保资源组织清晰合理：

```
AI_人工审阅结合工作框架资源包/
├── 01_使用说明文档/         # 资源包整体使用说明
├── 02_提示词模板/           # AI智能体工作提示词
│   └── 内容生成提示词/       # 内容生成类提示词
├── 03_工作规范文件/         # 工作标准和规范
├── 04_流程说明文档/         # 工作流程和协作机制
├── 05_辅助材料/             # 各类文档模板
└── 06_框架概述/             # 框架核心概念和设计原则
```

## 2. 内容整理与提取规则

### 2.1 提取原则
- **中文表述**：所有内容均采用中文表述
- **核心资源**：仅保留与管理框架展开相关的必要资源
- **排除项目特定信息**：删除项目具体工作执行细节、特属于本项目的经验总结
- **模块化**：将内容拆分为独立的模块化文件，便于AI智能体检索和使用
- **标准化**：统一文件命名和格式，确保一致性

### 2.2 提取内容范围

#### 01_使用说明文档
- 资源包使用说明
- 资源包组织结构说明
- AI智能体使用资源包的操作流程指导
- 常见问题解答

#### 02_提示词模板
- 提示词设计规范
- 内容生成提示词（网站爬取提示词）
- 质量检查提示词模板
- 问题修复提示词模板
- 报告生成提示词模板

#### 03_工作规范文件
- AI智能体工作规范
- 人工审阅标准
- 质量保证体系

#### 04_流程说明文档
- AI智能体工作流程
- 人工审阅流程
- AI与人工协作流程

#### 05_辅助材料
- 自检报告模板
- 人工审议意见表模板
- 修复报告模板

#### 06_框架概述
- 框架核心设计原则
- 框架架构

## 3. AI智能体角色定位与使用方式

AI智能体在使用该资源包时，应扮演**项目主管**角色，具体职责包括：

1. **需求理解**：通过多轮交互深入理解使用者的项目目标、需求边界及预期成果
2. **任务拆解**：将整体项目系统性拆解为可执行的任务模块与子任务
3. **任务规划**：明确各任务的优先级、依赖关系及时间节点
4. **提示词应用**：基于任务特性自动筛选并应用相匹配的提示词模板
5. **进度管理**：在任务执行过程中持续进行进度追踪、质量把控与风险识别

## 4. 资源包使用流程

1. **项目启动**：AI智能体通过使用说明文档了解资源包结构和使用方法
2. **需求分析**：AI智能体与使用者交互，深入理解项目需求
3. **任务规划**：AI智能体基于提示词模板拆解任务，制定执行计划
4. **执行监控**：AI智能体根据工作规范和流程指导任务执行
5. **质量控制**：AI智能体应用质量保证体系进行质量把控
6. **报告生成**：AI智能体使用报告模板生成各类报告
7. **持续改进**：AI智能体基于反馈优化工作流程和提示词

## 5. 资源包维护与更新机制

- 每季度审查一次资源包内容，确保与实际工作保持一致
- 根据业务需求和技术发展更新质量标准和工作流程
- 定期优化提示词模板，提高AI智能体的工作效率和质量

## 6. 预期成果

- 1个完整的AI智能体与人工审阅结合工作框架资源包
- 20+个模块化的资源文件
- 1份详细的使用说明文档
- 4类提示词模板
- 3类工作规范文件
- 3类流程说明文档
- 3类辅助材料模板
- 1份框架概述文档

该资源包将具备高度的可复用性和通用性，能够灵活适应不同业务场景和需求变化，支持AI智能体作为项目主管自主展开工作。
