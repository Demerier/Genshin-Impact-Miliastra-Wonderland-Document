# 原神千星奇域·综合指南 - 人工审议意见表

## 1. 目的与适用范围

### 1.1 目的
本审议意见表用于系统化记录人工审阅AI生成成果物的反馈与修改建议，确保审阅过程规范、透明，便于AI智能体理解和执行修复工作。

### 1.2 适用范围
适用于原神千星奇域·综合指南网站爬取项目中所有需要人工审阅的成果物，包括：
- Markdown文档（常规文档和分段文档）
- 目录文件
- 自检报告
- 进度报告
- 其他项目相关文件

## 2. 审议流程

### 2.1 审议准备

1. **获取待审议文件**：从指定目录获取需人工审议的文件
2. **了解审议标准**：熟悉成果物自检规范和质量标准
3. **准备审议工具**：准备必要的审阅工具（文本编辑器、浏览器等）
4. **明确审议重点**：根据文件类型和AI自检结果，确定审议重点

### 2.2 审议执行

```
┌─────────────────────────────────────────────────────┐
│ 1. 阅读文件内容，理解文件结构和主题               │
├─────────────────────────────────────────────────────┤
│ 2. 检查格式规范，确保符合Markdown标准             │
├─────────────────────────────────────────────────────┤
│ 3. 检查内容完整性和准确性                         │
├─────────────────────────────────────────────────────┤
│ 4. 检查链接有效性和正确性                         │
├─────────────────────────────────────────────────────┤
│ 5. 检查文件名和存储位置是否符合规范               │
├─────────────────────────────────────────────────────┤
│ 6. 记录发现的问题和建议                            │
├─────────────────────────────────────────────────────┤
│ 7. 评估文件质量，给出审议结果                     │
├─────────────────────────────────────────────────────┤
│ 8. 生成审议意见表                                 │
└─────────────────────────────────────────────────────┘
```

### 2.3 审议后处理

1. **提交审议意见**：将审议意见表保存到指定目录
2. **通知AI智能体**：AI智能体自动获取审议意见
3. **执行修复**：AI智能体根据审议意见修复问题
4. **验证修复结果**：人工或AI验证修复结果
5. **更新文件状态**：根据验证结果更新文件状态

## 3. 审议标准

### 3.1 内容质量审议标准

| 审议项 | 标准 | 严重程度判定 |
|--------|------|--------------|
| 完整性 | 文档内容完整，无缺失或截断 | 内容缺失>500字：严重<br>内容缺失100-500字：中等<br>内容缺失<100字：轻微 |
| 准确性 | 内容与原始页面一致，无错误或扭曲 | 关键信息错误：严重<br>次要信息错误：中等<br>排版错误：轻微 |
| 可读性 | 内容易读，段落清晰，格式规范 | 难以阅读：严重<br>阅读不便：中等<br>轻微影响：轻微 |
| 结构性 | 保持原始页面的标题层级和段落结构 | 结构混乱：严重<br>结构不清晰：中等<br>结构轻微问题：轻微 |
| 关联性 | 内容逻辑连贯，主题明确 | 逻辑混乱：严重<br>逻辑不清晰：中等<br>逻辑轻微问题：轻微 |

### 3.2 格式质量审议标准

| 审议项 | 标准 | 严重程度判定 |
|--------|------|--------------|
| 标题格式 | 使用正确的Markdown标题格式，层级清晰 | 标题格式错误>5处：严重<br>标题格式错误3-5处：中等<br>标题格式错误<3处：轻微 |
| 段落格式 | 每个段落单独一行，无连续空行 | 段落格式错误>10处：严重<br>段落格式错误5-10处：中等<br>段落格式错误<5处：轻微 |
| 列表格式 | 使用正确的Markdown列表格式，缩进统一 | 列表格式错误>5处：严重<br>列表格式错误3-5处：中等<br>列表格式错误<3处：轻微 |
| 代码块格式 | 使用 ``` 包裹代码块，语言标识正确 | 代码块格式错误>3处：严重<br>代码块格式错误1-3处：中等<br>无代码块：不适用 |
| 表格格式 | 使用正确的Markdown表格格式，表头与内容对齐 | 表格格式错误>3处：严重<br>表格格式错误1-3处：中等<br>无表格：不适用 |

### 3.3 链接质量审议标准

| 审议项 | 标准 | 严重程度判定 |
|--------|------|--------------|
| 内部链接 | 所有内部链接已转换为本地相对路径，指向的文件存在 | 无效链接>5处：严重<br>无效链接3-5处：中等<br>无效链接<3处：轻微 |
| 图片链接 | 所有图片已下载到本地images目录，链接指向的图片存在 | 无效图片链接>3处：严重<br>无效图片链接1-3处：中等<br>无效图片链接<1处：轻微 |
| 链接格式 | 链接格式正确（[链接文本](./文件名.md)） | 链接格式错误>5处：严重<br>链接格式错误3-5处：中等<br>链接格式错误<3处：轻微 |

### 3.4 文件名与存储审议标准

| 审议项 | 标准 | 严重程度判定 |
|--------|------|--------------|
| 命名规范 | 文件名符合命名规则，清晰反映文档内容 | 命名不符合规范：中等<br>命名不清晰：轻微 |
| 存储位置 | 存储在指定目录 | 存储位置错误：中等 |

### 8.2 审议注意事项

1. **保持客观**：基于事实和标准进行审议，避免主观判断
2. **详细记录**：清晰描述问题位置、类型和严重程度
3. **提供建议**：给出具体、可执行的修复建议
4. **注重效率**：根据文件类型和重要性合理分配审议时间
5. **及时反馈**：审议完成后及时提交审议意见表
6. **持续改进**：根据审议结果，持续优化审议标准和流程
